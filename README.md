# Representing-and-consulting-change-in-wikidata

This repository contains code and data for the empirical analysis and scalability evaluation of the MSc thesis **Representing and Consulting Change in Wikidata**.

The thesis studies how factual change is represented and queried in Wikidata. It analyzes two Wikidata domains, business entities and politicians, and evaluates the identified approaches in terms of expressiveness, usability, and scalability.

## Repository structure

- `business/`  
  Contains the code and data used for the approach analysis in the business domain, including domain pool construction, sampling, statement retrieval, property analysis, property taxonomy, qualifier taxonomy, change-related pair identification, and approach analysis.

- `politician/`  
  Contains the code and data used for the approach analysis in the politician domain, including domain pool construction, sampling, statement retrieval, property analysis, property taxonomy, qualifier taxonomy, change-related pair identification, and approach analysis.

- `scalability evaluation/`  
  Contains the query templates, input entity sets, QLever runtime logs, and summary files used for the scalability evaluation.

## Domain data

The domain pools were retrieved from the Wikidata Query Service (WDQS).

- Business entity pool: retrieved on 9 June 2026; 765,042 entities.
- Politician pool: retrieved on 28 April 2026; 929,221 entities.

From each pool, 15,000 entities were randomly sampled for the empirical analysis.

## Notes

Wikidata is continuously updated. The results in this repository reflect the data retrieved at the dates stated above. Runtime results also depend on the selected query platform and execution environment.

Some large intermediate files may not be included because of GitHub browser upload limits. These files can be regenerated by running the notebooks in the corresponding folders.
